I'm figuring out how algorithm speed works and my understanding is that O(1) speed is 1 cpu tick, like assigning stuff.

Then O(n) is one cycle or if you've looped over the data once. Then O(n2), n to the power of 2 is two cycles. So each cycle slows it down, meaning you want to optimize your algorithm to manipulate the data as little as it needs to to get the answer.

Brute forcing is when you check every single possible outcome, answer, essentially looping over everything instead of finding tricks to skip data. This where you first begin, before optimizing when you learn this stuff.
