Algorithmic speed-

O(n) = Speed constant with data. Linear increase. Looping, linked list searching.

O(1) = fastest you can get. Doesn't matter about data size.

O(log n) = Binary search. Fast.

O(n log n) = Slowish, quicksort, mergesort, heapsort. Slows down with larger datasets.

O(n^2) = Very slow, insertion sort, selection sort, bubblesort.

O(n!) = the slowest.
